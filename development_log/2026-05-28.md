#### VLM 验证分析

发现：GLM-4V-Plus (standard) 和 GLM-4.6V-Flash (detailed) 在验证 3D 零件时都倾向于返回 MISMATCH，即使模型描述准确。可能原因：

1. VLM 对 3D 模型的尺寸判断不够精确
2. 英文描述中的细节（如孔的数量、直径）在等轴测视图中不易验证
3. VLM 可能过于保守（宁可报 MISMATCH 也不误报 MATCH）

改进方向：
- 使用更简短的描述（只描述主要形状）
- 多角度验证（top + front + isometric）
- 使用更强大的 VLM 模型（如 GPT-4V）

### 技术发现

1. **VLM 定位精度**：`vlm_locate` 使用 "standard" detail 级别能准确定位菜单栏元素（坐标偏差 <5px），配合 `gui_click` 可实现可靠点击
2. **fc_menu vs fc_batch**：API 方式（fc_batch）精确快速，适合参数化建模；GUI 方式（fc_menu）视觉驱动，适合工作台特定功能。系统提示词中加入选择策略
3. **自动修复循环**：在 `_run_direct()` 中注入 user message 触发修复，Agent 自然地将其视为对话继续，无需修改模型调用逻辑
4. **VLM 验证保守性**：GLM-4V 系列在 3D 模型验证时倾向于返回 MISMATCH，需要更简洁的描述或更强的模型
5. **Web 面板实时性**：WebSocket 推送 + HTTP 轮询回退，FastAPI TestClient 可直接测试 API 端点
