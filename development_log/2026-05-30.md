### Task 12: DAG 任务依赖图 + 状态扩展

**修改文件：**
- `src/lang3d/agent/state.py` — PlanStep 增加 `dependencies: list[str]` 和 `assigned_agent: str`
- `src/lang3d/agent/dag.py` — **新建**

**dag.py 核心实现：**

```
TaskDAG
  ├─ add_step(step, dependencies, agent_role) → DAGNode
  ├─ get_ready_steps() → list[DAGNode]          # 所有依赖已完成的步骤
  ├─ mark_completed(step_id) → list[str]         # 返回新解锁的 step IDs
  ├─ mark_failed(step_id) → list[str]            # 级联标记依赖者为 SKIPPED
  ├─ parallel_groups() → list[list[DAGNode]]     # Kahn 算法拓扑排序 → 波次分组
  ├─ _detect_cycles()                             # DFS 检测环 → ValueError
  ├─ to_dict() → dict                            # 序列化给 Web 面板
  └─ from_plan(plan, assembly) → TaskDAG          # 从 Plan + Assembly 构建
```

- `parallel_groups()` 使用 Kahn 算法进行拓扑排序，将节点分为并行波次
- `from_plan()` 可从 `Assembly.joints` 推断依赖关系（child 依赖 parent）
- `agent_role` 根据步骤的 `expected_tools` 自动推断（modeling/vision/gui/general）

**state.py 变更：**
- PlanStep 新增 `dependencies` 和 `assigned_agent` 字段（默认空值，向后兼容）
- `save()`/`load()` 支持新字段序列化

**测试：** 21/21 通过 — DAG 构建、并行分组、就绪步骤、完成/失败标记、环检测、序列化、状态持久化

### Task 13: SubAgent + MessageBus + SharedToolRegistry

**新建文件：**

- `src/lang3d/agent/message_bus.py`
- `src/lang3d/agent/shared_registry.py`

**SubAgent 核心设计：**

```python
class SubAgentRole(str, Enum):
    MODELING = "modeling"      # 3D 建模专用
    VISION = "vision"          # VLM 视觉分析
    GUI = "gui"                # GUI 自动化
    VERIFICATION = "verification"  # 验证
    GENERAL = "general"        # 通用

class SubAgent:
    def execute(step, context) → SubAgentResult     # 同步执行
    async def execute_async(step, context)           # asyncio.to_thread 包装
```

- SubAgent 复用现有 `Executor` 类（已有的工具调用循环），不重复实现
- `context` 参数接收前置 Agent 结果，注入到步骤描述中传递给 LLM
- 每个 role 有专用的系统提示词
- `execute_async` 通过 `asyncio.to_thread()` 包裹同步方法，无需改造整个工具系统为 async

**MessageBus：**
- 发布订阅模式，跨 Agent 通信
- 支持按 sender 和 type 过滤消息
- `get_artifacts()` 自动收集所有文件路径

**SharedToolRegistry：**
- 用 `threading.Lock` 包装 `execute()` 方法
- 确保多线程并行时工具调用串行化

### Task 14: Assembly Verifier 装配验证

**新建文件：** `src/lang3d/agent/assembly_verifier.py`

**核心接口：**

```python
@dataclass
class FitCheck:
    joint: Joint; parent_part: Part; child_part: Part
    clearance: float; required_clearance: float; fits: bool; notes: str

class AssemblyVerifier:
    def verify_assembly(assembly, workspace, parts_results) → AssemblyVerificationResult
    def check_part_completeness(assembly, workspace) → list[PartCheck]
    def check_joint_fits(assembly) → list[FitCheck]
    def check_screw_holes(assembly) → list[str]
```

- 螺钉孔规格检查：对比零件尺寸与 `STANDARD_SCREWS` 标准
- 生成人类可读的装配报告

### Task 15: OrchestratorAgent 编排器

**新建文件：** `src/lang3d/agent/orchestrator.py`

**核心接口：**

```python
class OrchestratorAgent:
    def should_orchestrate(task, plan) → bool      # 启发式判断
    def run_task(task) → str                        # 主入口
    async def _run_orchestrated_async(task) → str
    async def _execute_wave(nodes) → dict           # asyncio.Semaphore(max_parallel)
    def _spawn_sub_agent(node) → SubAgent
    def _collect_context(node) → dict               # 收集依赖 Agent 结果
    def _handle_failure(node, result) → bool         # 反思+重试
```

**执行流程：**

```
run_task("设计一个3自由度机械臂")
  │
  ├─ create_plan() → TaskDAG.from_plan() → parallel_groups()
  │
  ├─ Wave 0: asyncio.gather([SubAgent(base_plate), SubAgent(servo_holder)])
  ├─ Wave 1: asyncio.gather([SubAgent(base_joint_housing)])  ← context from wave 0
  ├─ Wave 2: asyncio.gather([SubAgent(shoulder_link)])
  ├─ ...
  │
  └─ Final: AssemblyVerifier → 装配报告
```

**关键设计决策：**
- `asyncio.Semaphore(max_parallel=3)` 限制并发数
- 失败时调用 `Reflector.reflect()` + 重新执行（最多 3 次）
- `_collect_context()` 收集所有已完成依赖的结果，格式化为上下文注入
- `_try_get_assembly()` 自动匹配任务描述中的装配定义（如"机械臂"→ ROBOTIC_ARM_ASSEMBLY）

**测试：** 11/11 通过 — 编排决策、上下文收集、SubAgent 生成、装配匹配、完整运行

### 技术发现

1. **SubAgent 复用 Executor 而非重新实现**：Executor 已有完整的工具调用循环（LLM 交互 → 工具选择 → 执行 → 结果处理），SubAgent 只需传入不同系统提示词和上下文即可
2. **asyncio.to_thread 是同步→异步的正确桥接方式**：不需要改造整个工具系统为 async，`asyncio.to_thread(SubAgent.execute)` 将同步执行包装为协程
3. **Kahn 算法天然支持并行分组**：拓扑排序过程中每轮移除所有入度为 0 的节点，这些节点就是可并行执行的波次
4. **上下文传递方式**：将前置 Agent 结果注入到步骤描述中（而非独立的 messages），因为 Executor.execute_step() 自行构建 messages
5. **e2e 脚本与 pytest 的冲突**：以 `test_` 开头的函数名会被 pytest 自动收集，e2e 脚本应避免使用 `test_` 前缀的辅助函数
