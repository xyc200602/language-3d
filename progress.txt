## 2026-06-04 - Task 40: 迭代设计优化

### What was done:
- 创建 `src/lang3d/tools/iteration.py` — 迭代设计优化工具
- 支持 5 种变更类型：payload（负载）、reach（工作半径）、material（材料）、actuator（执行器）、controller（控制器）
- `analyze_impact()` — 分析需求变更的影响范围（影响哪些零件/关节/制品）
- `apply_change()` — 应用变更并只重新生成受影响的制品（增量更新）
- `iterate_design()` — 完整迭代流程（分析→应用→重新生成）
- `compare_designs()` — 比较两个设计快照的差异
- `create_snapshot()` — 创建设计快照，可选生成全部制品
- `IterationTool` — Agent 工具注册
- 注册到 `agent/core.py` 和 `tools/base.py`

### 关键设计决策:
- 变更严重程度自动评估：minor / moderate / major（基于变化比例）
- 增量更新：只重新生成受影响制品（如材料变更不触发固件重生成）
- 深拷贝保护：原始快照在迭代中不被修改
- 负载变更自动重新选型：如果当前执行器力矩不足，自动选择更强的型号

### Testing:
- 53/53 测试通过（test_iteration.py）
- 覆盖：影响分析、变更应用、增量更新、设计比较、完整迭代、工具执行
- 0 回归（e2e_design 16/16 仍通过）

## 2026-06-04 - Task 37: 3D 打印优化

### What was done:
- **新建 `tools/print_optimize.py` — 3D 打印优化工具**
  - `optimize_orientation(part)` — 打印方向推荐
    - 圆柱体竖直放置（轴竖直，最佳层粘合）
    - 方形体平放（最小高度在 Z 方向，减少层数）
    - 检测特征：孔/轴承座/走线通道 → 调整方向
    - 标记是否需要支撑材料
  - `apply_tolerance_compensation(dims, fit_type, material)` — 公差补偿
    - 收缩补偿：PLA 0.3%, ABS 0.8%, PETG 0.4%, NYLON 1.2%
    - 配合间隙：tight_fit 0.15mm, sliding_fit 0.3mm, loose_fit 0.5mm, bearing_fit 0.1mm
    - 外部尺寸减间隙，内部尺寸加间隙
  - `recommend_print_params(part, quality)` — 参数推荐
    - 3 级质量：draft (0.3mm/10%), standard (0.2mm/20%), high (0.12mm/30%)
    - 轴承座零件自动增加填充 40% 和壁厚 4 层
    - 轻量化零件自动降低填充 15%
    - 粗略估算打印时间
  - `pack_parts_on_plate(parts, printer)` — 批量排版
    - 贪心网格排列，最大面积优先
    - 支持 Ender3/Prusa MK3/Bambu X1C/自定义
    - 计算排版利用率和所需板数
  - `optimize_assembly_print(assembly, ...)` — 一站式优化
  - `PrintOptimizeTool` — Agent 工具

- **修改 `agent/core.py`** — 注册 print_optimize_tools

完成了 Phase 2 全部任务，实现了多 Agent 编排能力：将复杂任务分解为 DAG 依赖图，用多个轻量 SubAgent 并行执行独立步骤，最后进行装配验证。同时修复了 5 个 e2e 测试脚本的 pytest 收集问题。总测试数从 115 增长到 191，全部通过。

#### TaskDAG 依赖图 (`agent/dag.py` 新建)

```
TaskDAG
  ├─ add_step(step, dependencies, agent_role) → DAGNode
  ├─ get_ready_steps() → list[DAGNode]          # 所有依赖已完成的步骤
  ├─ mark_completed(step_id) → list[str]         # 返回新解锁的 step IDs
  ├─ mark_failed(step_id) → list[str]            # 级联标记依赖者为 SKIPPED
  ├─ parallel_groups() → list[list[DAGNode]]     # Kahn 算法拓扑排序 → 波次分组
  ├─ _detect_cycles()                             # DFS 检测环 → ValueError
  ├─ to_dict() → dict                            # 序列化给 Web 面板
  └─ from_plan(plan, assembly) → TaskDAG          # 从 Plan + Assembly 构建
```

- `parallel_groups()` 使用 Kahn 算法进行拓扑排序，将节点分为并行波次
- `from_plan()` 可从 `Assembly.joints` 推断依赖关系（child 依赖 parent）
- `agent_role` 根据步骤的 `expected_tools` 自动推断（modeling/vision/gui/general）

`state.py` 变更：PlanStep 新增 `dependencies` 和 `assigned_agent` 字段（默认空值，向后兼容）

### 视觉模型评估 + 多模型路由

**GLM 视觉模型全面评估**：

| 模型 | Coding Plan API | 标准 API | 备注 |
|------|----------------|----------|------|
| GLM-4V-Flash | 可用 | 可用 | 免费 |
| GLM-4V | 可用 | 可用 | 收费 |
| GLM-4V-Plus | 可用 | 可用 | 收费，质量最好 |
| GLM-4.6V | 返回空内容 | 可用 | 需标准 API |
| GLM-4.6V-Flash | 返回空内容 | 可用 | 需标准 API |
| GLM-5V-Turbo | 429 限权 | **可用** | 需标准 API |

**GLM-4.6V "返回空内容" 排查**：不是模型不能用，而是之前 `max_tokens=500` 不够。GLM-4.6V 输出以 `\n` 开头 + `finish_reason=length` 截断 = 看起来像空内容。实际上需要 `max_tokens >= 2048` 才能得到完整回答。

**max_tokens 上限测试**：

| 模型 | 实际 API 上限 | 最佳设置 |
|------|-------------|---------|
| GLM-4V-Flash | 1024 | 1024 |
| GLM-4V-Plus | 2048 | 2048 |
| GLM-4.6V-Flash | 16384+ | 16384 |
| GLM-4.6V | 16384+ | 16384 |

**视觉模型路由实现** — 4 级路由配置：

| Level | 模型 | max_tokens | 速度 | 适用场景 |
|-------|------|-----------|------|---------|
| `fast` | GLM-4V-Flash | 1024 | ~1s | 快速识别（颜色、形状） |
| `standard` | GLM-4V-Plus | 2048 | ~3s | 精准分析（UI 识别） |
| `detailed` | GLM-4.6V-Flash | **16384** | ~20s | CAD 验证（详细描述） |
| `maximum` | GLM-4.6V | **16384** | ~40s | 复杂检测（最全面） |

**改动的文件**：
- `models/base.py`：vision() 默认 max_tokens 2048→4096
- `models/glm.py`：默认 max_tokens 1024→4096；新增 `alt_client`（标准 API）；`VISION_MODEL_CAPS`
- `models/router.py`：新增 `VisionDetail` 枚举（4 级）；`VISION_DETAIL_MODELS` 路由表
- `tools/vlm.py`：4 个工具全部新增 `detail` 参数，默认 `detailed`

**Testing**: 38/38 单元测试通过；8/8 VLM 感知 e2e 测试通过；4/4 视觉路由级别验证通过

**关键技术发现**：
1. **双 API 端点策略**：Coding Plan API (`/coding/paas/v4`) 不支持 GLM-4.6V 系列和 GLM-5V-Turbo，需要 Standard API (`/api/paas/v4`)。GLMBackend 自动根据模型名选择端点
2. **max_tokens 是视觉模型可用性的关键**
3. **多模型路由的经济性**：简单任务用免费的 Flash，CAD 验证用详细的 4.6V-Flash
